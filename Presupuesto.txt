 Se realizó una entrevista con el cliente y obtuvimos la siguiente conclusión: El cliente quiere una pagina para dar a conocer su alojamiento por días, mostrar fotos del departamento y lugares cercanos, consultar disponibilidad y hacer reservas.
La competencia del cliente usan paginas como booking donde se ofrecen diferentes lugares, y se comparan entre ellos. El objetivo del cliente sería poder cobrarle al usuario un precio menor al que se cobraría contratando el servicio por booking evitando pagar
la comisión que esa pagina cobra. Los obstáculos para conseguir el objetivo es de que manera cobrarle al usuario, para no tener que usar una pasarela de pago externa que implique pagar una comisión extra que es lo que inicialmente se trataría de evitar;
otro obstaculo sería que la pagina tenga una buena visibilidad para poder llegar a los clientes. Otro obstaculo es que el costo de mantenimiento de la pagina sea mayor a la ganancia que se obtiene de ahorrarse la comisión.
 Del analisis del alcance del proyecto el cliente va a necesitar mantener datos de los clientes que harian una reserva, una base de datos para guardar dichos datos, la pasarela de pago no estaría incluida en el proyecto, fotos del lugar y hosting para la pagina web, que no estaría incluido dentro del servicio.
Para la realización del proyecto se usara HTML con estilos con CSS, se usara Bootstrap como framework para el desarrollo de la pagina, se usara sass para preprocesar el css. El proyecto se mantendra en github y se haran 4 entregas
donde en cada entrega se mostrara una versión y se realizara una entrega del 28%, 43%, 14% y 15% del pago por entrega respectivamente de un total de 350 dolares. En cada entrega se realizará una reunion con el cliente para mostrar los avances de cada entrega y recibir feedback sobre posibles cambios de estilos,
cambios sobre nuevas funcionalidades o secciones serán posibles pero no estaran incluidos dentro del presupuesto, teniendo un costo extra definido luego de un analisis de posibilidad de agregarlo y tiempo que llevaría.

 Para la realización del proyecto se tomaran 7 días habiles. En una primera entrega se entregara una pagina navegable, con un estilo basico, que luego se adaptara en siguientes entregas para esto se tomaran 2 días. En la segunda entrega se agregaran efectos, fondos y animaciones si las tuviera,
y se personalizaran estilos en los componentes para esta entrega se tomaran 3 días. En la tercera entrega, se agregara un SEO basico a la pagina y se haran ajustes sobre lo entregado, se tomara 1 día. Por último se hara una ultima entrega para hacer los ultimos ajustes y la presentación final se tomara 1 día.
 
 No se incluye la subida de la pagina al host ni costos relacionados como dominio, certificado ssl, dominio, etc. No se incluye desarrollo back-end del proyecto, pagina dinamica con respuestas en botones ni mantenimiento de la pagina. Todo lo que sea contenido de texto y multimedía debe ser entregado por el cliente.
Todos los recursos de diseño de marca deben ser brindado por el cliente.
